# 방향성 비순환 그래프 (DAG) 구현 가이드

기하학 퍼즐(Euclidea)에서 도형들은 서로 긴밀하게 연결되어 있습니다. 두 선이 교차하여 점이 생기고, 그 점을 중심으로 원이 그려지며, 그 원이 다시 다른 선과 만나 새로운 점을 만듭니다. 
따라서 부모 도형(예: 선)을 지우면, 그 자식 도형(예: 교차점과 원)들도 연쇄적으로 지워져야 합니다. 

이러한 종속성(Dependency)을 관리하기 위해 우리 프로젝트는 **방향성 비순환 그래프 (Directed Acyclic Graph, DAG)** 구조를 도입했습니다.

## 1. 어떻게 구현되었나요? (구현 방법)

DAG 구조는 객체 지향적인 포인터(메모리 참조) 방식이 아닌, **고유 ID 기반의 관계 매핑** 방식을 통해 매우 가볍게 구현되었습니다. 

### A. 모든 엔티티에 고유 ID 부여 (`src/entities.ts`)
그래프의 노드(Node) 역할을 하기 위해, 보드 위에 올라가는 모든 `Point`, `Line`, `Circle`은 생성될 때 `id` 속성을 갖게 됩니다.
외부 패키지(uuid) 의존성 문제를 피하기 위해 내부적으로 숫자가 1씩 증가하는 카운터를 사용하여 `entity_1`, `entity_2`와 같은 고유한 문자열 ID를 자동으로 생성합니다.

### B. 부모를 기억하는 배열 추가 (`parents: string[]`)
모든 엔티티는 자신이 누구로부터 파생되었는지 기억하는 간선(Edge) 정보를 갖습니다. 이 정보가 바로 `parents` 배열입니다.

* **직접 그린 도형 (Tools):** 사용자가 `LineTool`로 점 A와 점 B를 클릭해 선을 그렸다면, 새롭게 생성된 선 객체는 `parents: [점A의 ID, 점B의 ID]`를 갖게 됩니다.
* **자동 생성된 교차점 (Board):** 사용자가 두 개의 원을 그렸을 때 엔진이 수학적으로 교차점을 찾아내면, 엔진은 새로운 `Point`를 생성하며 그 점의 부모로 `parents: [원1의 ID, 원2의 ID]`를 기록합니다.

이 단순한 1차원 배열(`parents`)만으로도 완벽한 단방향 그래프가 형성됩니다. 자식은 부모를 알지만, 부모는 자식을 모릅니다. (방향성) 
그리고 기하학 작도의 특성상, 미래에 그려진 도형이 과거의 도형의 부모가 될 수 없으므로 절대 순환(Cycle)이 발생하지 않습니다. (비순환)

## 2. 연쇄 삭제 (Recursive Deletion)의 작동 원리

이 DAG 구조가 가장 빛을 발하는 곳이 바로 **지우개(Eraser)** 로직입니다.
`src/board.ts` 내의 `removeEntityById(id)` 함수는 다음과 같은 순서로 재귀적(Recursive)인 삭제를 수행합니다.

1. **타겟 삭제:** 넘겨받은 `id`를 가진 도형을 보드의 배열에서 제거합니다.
2. **자식 탐색:** 보드에 남아있는 모든 점, 선, 원을 하나씩 검사합니다. 만약 어떤 도형의 `parents` 배열에 **방금 지워진 `id`가 포함되어 있다면?**
3. **재귀 호출:** 그 자식 도형 역시 지워져야 하므로, 자식 도형의 ID를 가지고 자기 자신(`removeEntityById(child_id)`)을 다시 호출합니다.
4. 이 과정은 더 이상 지워진 부모를 의존하는 자식이 없을 때까지 연쇄적으로(폭포수처럼) 실행됩니다.

## 3. 고아(Orphan) 청소 기능

부모를 지우면 자식이 연쇄적으로 지워지는 것 외에도, 반대로 **자식을 지웠을 때 쓸모없어지는 부모**를 치우는 역방향 로직도 포함되어 있습니다.

허공에 원을 그리면 `원(자식)`과 `점 두개(부모)`가 생성됩니다. 사용자가 지우개로 이 원을 지우면, 자식인 원만 지워지고 허공에 아무 쓸모없는 부모 점 두 개가 덩그러니 남게 됩니다. 

이를 방지하기 위해 `removeEntityById` 함수의 마지막 단계에 **고아 청소(Orphan Cleanup)** 로직이 들어있습니다:
* 방금 지우기 연산이 모두 끝난 보드를 스캔하여, 아무런 선이나 원에서도 부모(`parents`)로 참조하지 않고 있는 잉여 점(Point)이 있다면 그 점을 화면에서 조용히 삭제합니다.

## 요약
우리 엔진은 무거운 그래프 라이브러리를 쓰지 않습니다. 그저 모든 도형에 `id`와 `parents` 배열을 달아주는 것만으로 완벽한 종속성 그래프를 구현했으며, 이를 통해 실제 기하학의 법칙에 맞는 완벽한 지우개 연쇄 삭제를 달성했습니다.
